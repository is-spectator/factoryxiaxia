# 0319 人工验收脚本

适用分支：`codex/agent-platform-phase2`

本轮验收目标：
- 验证官方数字员工从搜索、购买、支付、部署到 API 交付的完整链路
- 验证官方执行型员工“空空”从购买、实例化到 OpenClaw 工作台交付的完整链路
- 验证新增的“我的机器人”管理台是否可正常使用
- 验证普通员工老流程没有被破坏

---

## 一、验收前准备

### 1. 环境准备

确认以下内容已经就绪：

1. 前端页面可访问
2. 后端 API 可访问
3. 数据库已初始化
4. 测试账号可正常登录
5. 系统中存在以下官方数字员工：
   - `多语言客服 #12`
   - `公众号运营官 #18`
   - `空空`

### 2. 推荐验收环境

建议：

1. 使用一个全新的普通用户账号
2. 使用桌面端浏览器执行一次完整流程
3. 开启浏览器开发者工具，必要时查看网络请求

---

## 二、主流程验收

### 场景 A：官方客服机器人购买与部署

#### 步骤 1：进入市场页

操作：

1. 打开 `workers.html`

预期结果：

1. 页面顶部可见“官方数字员工专区”
2. 页面有“只看官方机器人”按钮
3. 官方专区至少展示 2 个官方机器人

#### 步骤 2：筛选官方机器人

操作：

1. 点击“只看官方机器人”

预期结果：

1. 员工列表只显示官方机器人
2. 列表卡片带有“官方”或“可部署 API”等标识

#### 步骤 3：查看官方机器人详情

操作：

1. 点击 `多语言客服 #12`

预期结果：

1. 详情页显示“官方数字员工”标识
2. 右侧展示套餐，不展示小时租赁输入框
3. 页面显示适用场景、交付流程、官方模板说明
4. 主按钮文案为“购买并开始部署”

#### 步骤 4：创建订单

操作：

1. 选择任意套餐
2. 点击“购买并开始部署”

预期结果：

1. 跳转到 `order-create.html`
2. 页面显示“套餐与交付配置”
3. 费用明细显示套餐价格和服务周期
4. 提交按钮文案为“确认购买”

#### 步骤 5：支付订单

操作：

1. 点击“确认购买”
2. 根据当前环境执行支付：
3. `development` 环境：使用“模拟支付”并点击“确认支付”
4. `production` 环境：提交付款确认后，由管理员在后台确认收款

预期结果：

1. 订单状态变为 `paid`
2. 页面出现“开始部署”入口
3. 进入 `deployment-setup.html`

#### 步骤 6：创建部署实例

操作：

1. 在部署向导填写实例名称
2. 可选填写组织名称
3. 点击“创建部署实例”

预期结果：

1. 创建成功
2. 页面显示实例状态，不再是“未创建”
3. 页面能看到组织、机器人、套餐等基础信息

#### 步骤 7：保存部署配置

操作：

1. 填写企业名称
2. 选择品牌语气
3. 填写禁答话题和敏感关键词
4. 点击“保存配置”

预期结果：

1. 保存成功提示出现
2. 刷新页面后配置仍保留

#### 步骤 8：上传知识库

操作：

1. 点击“填入示例内容”或手工填写知识文档
2. 点击“上传知识文档”

预期结果：

1. 上传成功提示出现
2. 知识库区域出现至少 1 条知识库记录
3. `document_count` 大于 0

#### 步骤 9：发布机器人

操作：

1. 点击“发布机器人”

预期结果：

1. 发布成功提示出现
2. 部署状态变为 `active`
3. 页面显示 `public_token`
4. 页面显示公开 API endpoint
5. 页面显示 `cURL` 和前端调用示例

#### 步骤 10：验证公开 API

操作：

1. 复制部署页中的 endpoint 或 `cURL` 示例
2. 发送一条测试消息

请求示例：

```bash
curl -X POST 'http://localhost/api/public/chat/<public_token>/message' \
  -H 'Content-Type: application/json' \
  -d '{
    "message": "你好，我想咨询退款政策",
    "visitor_name": "官网访客"
  }'
```

预期结果：

1. 返回 HTTP 200
2. 返回 JSON
3. 返回体里包含机器人回复
4. 回复不是空字符串

## 三、空空专项验收

### 场景 B：空空购买与 OpenClaw 工作台交付

#### 步骤 1：进入空空详情页

操作：

1. 打开 `workers.html`
2. 开启“只看官方机器人”
3. 点击 `空空`

预期结果：

1. 详情页展示“空空 OpenClaw Workspace”
2. 页面可见 `99/月` 和 `2 元/小时` 两种定价
3. 页面文案明确说明“独立隔离实例”与 `OpenClaw` 工作台交付
4. 主按钮文案为“购买并创建空空实例”

#### 步骤 2：创建空空订单

操作：

1. 选择月付或小时包
2. 点击“购买并创建空空实例”
3. 在下单页确认方案并提交

预期结果：

1. 成功生成 `agent_deployment` 类型订单
2. 订单详情里可见 `空空`
3. 支付后会进入部署向导，而不是普通知识库发布流程

#### 步骤 3：完成支付

操作：

1. 根据当前环境完成支付
2. 支付完成后进入 `deployment-setup.html`

预期结果：

1. 部署页标题和步骤文案是“空空 / OpenClaw 工作台”模式
2. 页面不显示知识库上传和 `public_token` API 交付卡片
3. 页面显示实例名称、模型和运行时信息

#### 步骤 4：创建空空实例

操作：

1. 填写实例名称
2. 可选调整模型和运行参数
3. 点击“创建部署实例”

预期结果：

1. 创建成功
2. 自动生成 `kongkong_instance`
3. 部署状态进入 `active` 或实例状态进入 `running`
4. 页面显示工作台入口链接

#### 步骤 5：验证工作台入口

操作：

1. 点击“进入空空工作台”
2. 如页面提示需要 `Gateway Token`，使用页面弹窗提示或实例详情中的 token

预期结果：

1. 新窗口成功打开 OpenClaw
2. 页面标题包含 `OpenClaw Control`
3. 页面不是空白页，不返回 404 / 502
4. 实例实际使用的模型为 `dashscope / qwen-max`

#### 步骤 6：验证我的机器人中的空空实例

操作：

1. 打开 `my-agents.html`
2. 选择空空实例
3. 尝试点击“进入工作台”
4. 依次测试“暂停实例”“启动实例”或“重启实例”

预期结果：

1. 空空实例显示独立状态、入口链接和运行信息
2. 点击“进入工作台”仍可打开 OpenClaw
3. 暂停后状态变为 `suspended`
4. 重新启动后状态恢复为 `running`

#### 步骤 7：验证用户隔离

操作：

1. 使用用户 A 购买并创建一份空空
2. 使用用户 B 再购买并创建一份空空
3. 分别记录两份实例的容器名、实例名或入口链接

预期结果：

1. 两个用户得到的是两份不同实例
2. 两份实例的 `instance_slug / container_name / gateway_token / entry_url` 不相同
3. 用户 A 无法在前台直接管理用户 B 的空空实例

## 四、我的机器人管理台验收

### 场景 C：已购机器人进入管理态

#### 步骤 1：进入“我的机器人”

操作：

任选以下一种方式：

1. 顶部导航点击“我的机器人”
2. 个人中心点击“我的机器人”
3. 从部署向导返回“我的机器人”

预期结果：

1. 能进入 `my-agents.html`
2. 页面展示机器人列表和管理台标题

#### 步骤 2：查看机器人列表

操作：

1. 观察左侧机器人列表
2. 点击不同机器人实例

预期结果：

1. 列表能显示实例名称、状态、套餐、渠道、到期时间
2. 当前选中项有高亮态
3. 点击切换后右侧详情同步变化

#### 步骤 3：查看运行数据

操作：

1. 观察顶部统计卡片
2. 观察右侧当前实例统计

预期结果：

1. 顶部能看到机器人实例数、运行中机器人、累计消息、平均置信度
2. 当前实例能看到会话总数、累计消息、待处理转人工、知识命中

#### 步骤 4：查看知识库

操作：

1. 查看“知识库概览”

预期结果：

1. 能看到刚才上传的知识库
2. 能看到文档总数、已发布数、发布时间
3. “去维护知识库”可跳回部署向导

#### 步骤 5：查看最近会话

操作：

1. 查看“最近会话”区域

预期结果：

1. 如果已有会话，能看到最近消息和机器人回复
2. 如果没有会话，会显示空状态提示

#### 步骤 6：控制台测试消息

操作：

1. 在“控制台测试”输入消息
2. 点击“发送测试消息”

建议内容：

```text
你好，请告诉我退款政策和人工客服处理时效。
```

预期结果：

1. 发送成功提示出现
2. 页面显示“最近一次测试回复”
3. “最近会话”区域同步出现新的会话或消息

#### 步骤 7：复制交付信息

操作：

1. 点击“复制 Token”
2. 点击“复制 Endpoint”

预期结果：

1. 页面显示“已复制到剪贴板”
2. 剪贴板内容与页面展示一致

## 七、通过标准

以下条件全部满足，才算本轮人工验收通过：

1. 官方机器人购买链路完整可用
2. 部署向导可创建实例、保存配置、上传知识、发布成功
3. 发布后公开 API 可以返回回复
4. 空空购买后可成功交付 OpenClaw 工作台入口
5. 空空工作台能真实打开，且实例模型为 `qwen-max`
6. “我的机器人”页面可以展示部署、知识库、会话、空空实例和指标
7. 控制台测试消息可正常工作
8. 导航和入口都能正确跳转
9. 普通员工旧流程未被破坏
