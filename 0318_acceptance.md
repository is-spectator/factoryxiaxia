# 0318 前端验收脚本

适用分支：`codex/agent-platform-phase2`

适用范围：官方数字员工专区、套餐购买、支付后部署向导、发布后 API 交付。

## 一、验收前准备

1. 启动项目，确保前后端可访问。
2. 准备一个普通用户账号。
3. 确认数据库里已初始化出 2 个官方数字员工：
   - `多语言客服 #12`
   - `公众号运营官 #18`
4. 本轮支付使用“模拟支付”。

## 二、主流程验收

### 1. 市场页验收

操作：
- 打开 `workers.html`
- 查看顶部是否出现“官方数字员工专区”
- 点击“只看官方机器人”

预期结果：
- 页面能展示官方专区卡片
- 至少能看到 `多语言客服 #12` 和 `公众号运营官 #18`
- 筛选后列表仅剩 `agent_service` 类型机器人

### 2. 详情页验收

操作：
- 点击 `多语言客服 #12`

预期结果：
- 页面出现“官方数字员工”标识
- 右侧不是“小时租赁”，而是套餐选择
- 能看到交付说明、适用场景、部署流程
- 按钮文案为“购买并开始部署”

### 3. 下单页验收

操作：
- 在详情页选择任意套餐
- 点击“购买并开始部署”

预期结果：
- 进入 `order-create.html`
- 页面展示的是套餐与交付配置，不是小时数输入框
- 费用明细显示套餐价格和服务周期
- 提交按钮文案为“确认购买”

### 4. 支付页验收

操作：
- 点击“确认购买”
- 进入支付页后使用“模拟支付”

预期结果：
- 支付页能看到套餐信息
- 支付成功后弹窗按钮显示“开始部署”
- 3 秒内自动跳转到 `deployment-setup.html`

### 5. 部署向导验收

操作：
- 在部署向导填写实例名称
- 可选：新建组织
- 点击“创建部署实例”
- 填写品牌语气、禁答话题、敏感关键词
- 点击“保存配置”

预期结果：
- 创建部署实例成功
- 页面显示当前部署状态，不再是“未创建”
- 保存配置成功，刷新后配置仍在

### 6. 知识库上传验收

操作：
- 在部署向导点击“填入示例内容”或手工填写一篇文档
- 点击“上传知识文档”

预期结果：
- 上传成功提示出现
- 知识库区域出现至少 1 条知识库卡片
- 卡片中的 `document_count` 大于 0

### 7. 发布与 API 交付验收

操作：
- 点击“发布机器人”

预期结果：
- 发布成功
- 部署状态变为 `active`
- 页面显示 `public_token`
- 页面显示公开 API endpoint
- 页面显示 `cURL` 和前端调用示例

## 三、API 可用性验收

操作：
- 复制部署向导里的 endpoint 和 `cURL` 示例
- 发起一次聊天请求

请求示例：

```bash
curl -X POST 'http://localhost/api/public/chat/<public_token>/message' \
  -H 'Content-Type: application/json' \
  -d '{
    "message": "你好，我想咨询退款政策",
    "visitor_name": "官网访客"
  }'
```

预期结果：
- 返回 `200`
- 返回 JSON 中包含回复内容
- 回复内容不是空字符串

## 四、订单页回归验收

### 1. 订单列表

操作：
- 打开 `orders.html`

预期结果：
- 官方机器人订单显示“官方机器人”标识
- 已支付/已激活的官方机器人订单带有“部署向导”入口

### 2. 订单详情

操作：
- 打开刚才生成的订单详情页

预期结果：
- 页面有“机器人部署”区块
- 区块能看到套餐、服务周期、交付形态
- 页面有“进入部署向导”或“管理机器人”按钮

## 五、回归检查

再额外确认 3 件事：

1. 普通员工仍然走原来的小时租赁流程，不受这轮改动影响。
2. 官方机器人支付成功后，不再跳普通订单详情，而是跳部署向导。
3. 已发布机器人重新进入部署向导时，能看到已有配置、知识库和 API 信息。

## 六、通过标准

以下 7 项全部满足，才算本轮验收通过：

1. 官方专区可见且筛选可用。
2. 官方详情页展示套餐购买，不展示小时租赁。
3. 官方机器人下单成功并能进入支付页。
4. 支付成功后自动跳部署向导。
5. 部署实例可创建，配置可保存。
6. 知识文档可上传，机器人可发布。
7. 发布后能拿到 `public_token`，公开聊天 API 可正常返回结果。

## 七、常见失败点

1. 看不到官方专区：
   - 检查数据库是否已初始化官方数字员工。
2. 支付后没有跳部署向导：
   - 检查订单 `order_type` 是否为 `agent_deployment`。
3. 发布失败：
   - 通常是没有上传知识文档。
4. API 没有返回：
   - 检查是否已经发布成功，以及 `public_token` 是否来自当前部署实例。
